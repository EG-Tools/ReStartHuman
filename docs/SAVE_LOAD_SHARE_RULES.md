# SAVE_LOAD_SHARE_RULES.md

This file describes save-slot persistence, loading behavior, and image sharing ownership.

## 1) Main owners
### Save/load
- Hook: `src/hooks/useSaveSlots.ts`
- Storage helpers: `src/utils/saveSlots.ts`
- Modal UI: `src/components/result/SaveSlotModal.tsx`
- Shared modal helpers: `src/components/result/saveSlotModal.shared.ts`

### Share image
- Hook: `src/components/result/useResultShare.ts`
- Result capture owner: `src/components/result/ResultScreen.tsx`

## 2) Save-slot model
Current type:
- `SaveSlotRecord` in `src/types/retireCalc.ts`

Current fields:
- `slotId`
- `name`
- `savedAt`
- `formData`
- `result`

Current storage key pattern:
- `kr-retire-calc-slot-{slotId}`

Current slot count:
- 5

Current storage version:
- `SAVE_SLOT_STORAGE_VERSION = 1`

## 3) Save/load behavior
### Saving
- Saving happens through `useSaveSlots().saveSlot()`.
- The app writes a full record to local storage.
- Slot name is normalized before persistence.

### Loading
- Loading applies `slot.formData` back into app state.
- `App.tsx` merges loaded data with `defaultFormData` before using it.
- After loading, the app opens the result route and recalculates from current logic.

This means the stored `formData` is the real source for resumed behavior.
Stored `result` exists in the record shape, but the app still recalculates when opening the loaded scenario.

## 4) Name rules
Defined in `normalizeSaveSlotName()`.

Current behavior:
- collapse repeated spaces
- trim leading/trailing spaces
- cap to 24 characters
- treat `빈슬롯` and autogenerated names like `은퇴계산3` as empty names

## 5) Migration / invalid-data rules
Defined in `readSaveSlotRecords()` and `migrateParsedSaveSlotRecord()`.

Current behavior:
- non-JSON data is ignored
- records with an unknown `version` are ignored
- malformed shape is ignored
- slots are sorted by slot number before use

If storage structure changes, update the migration path and tests in the same task.

## 6) What to be careful about
- The current save record includes both `formData` and `result`.
- The app currently reloads based on `formData`, so stored `result` should not be treated as stronger than the current calculation engine.
- If you change `RetireCalcFormData`, check whether old save data needs migration help.
- If you change slot-name rules, update modal behavior and tests together.

## 7) Share image notes
- Sharing is built around the captured result-screen content.
- `ResultScreen.tsx` owns the capture ref and passes it to the share hook.
- UI marked with `data-capture-exclude="true"` is intentionally excluded from the exported image.

If the screenshot/exported image is wrong, start in:
- `src/components/result/ResultScreen.tsx`
- `src/components/result/useResultShare.ts`
